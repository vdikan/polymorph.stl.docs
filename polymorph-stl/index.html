Polymorph Standard Library

Attempt to document libraries of the polymorph.stl project.

Project Overview

polymorph.stl is a Common Lisp standard library implementation initiative lead by @commander-trashdin. It is based upon @digicar99's polymorphic-functions.

This documentation is built with the fork of 40ants-doc system by @svetlyak40wt that is extended to allow for 'polymorphic-function symbols' documentation export.

Arithmetics

Basic Arithmetic Functions

Ad-hoc polymorphic realisation of basic arithmetic functions: +, -, * and /.

polymorphic-function
polymorph.maths:+
&rest xs

Arithmetic addition.

Polymorphs:

(+)

When called without arguments, '+ returns 0.

(+ T)

When called with only one argument A of any type, return A.

(+ NUMBER NUMBER)

Two arguments of type NUMBER are added with CL:+ .

(+ CHARACTER CHARACTER)

Addition of CHARACTER type arguments results in CHARACTER with the code as the sum if the arguments' codes.

(+ T T T &REST)

Reduce summation over multiple arguments.

Example of CHARACTER summation:

(+ #\a #\5 #\t) ; => #\LATIN_CAPITAL_LETTER_C_WITH_DOT_ABOVE

(format nil "~a" (+ #\a #\5 #\t)) ; => ÄŠ
polymorphic-function
polymorph.maths:-
x &rest xs

Arithmetic subtraction.

Polymorphs:

(- NUMBER)

Unary negation for NUMBER A, same as CL:- .

(- NUMBER NUMBER)

Subtract NUMBER SECOND from NUMBER FIRST.

(- CHARACTER CHARACTER)

Subtraction of CHARACTER type arguments results in CHARACTER with the

code as the subtraction of the CHARACTER SECOND from CHARACTER FIRST.

(- T T T &REST)

Reduce subtraction over multiple arguments.

polymorphic-function
polymorph.maths:*
&rest xs

Arithmetic multiplication.

Polymorphs:

(*)

When called without arguments, '* returns neutral element 1.

(* T)

When called with only one argument A of any type, return A.

(* NUMBER NUMBER)

Multiply NUMBER FIRST and NUMBER SECOND.

(* T T T &REST)

Reduce multiplication over multiple arguments.

polymorphic-function
polymorph.maths:/
x &rest xs

Arithmetic division.

Polymorphs:

(/ NUMBER)

Inverse of NUMBER A, same as CL:/ .

(/ NUMBER NUMBER)

Divide NUMBER FIRST with NUMBER SECOND.

(/ T T T &REST)

Reduce division over multiple arguments.

Updating Macros

The updating macros corresponding to the functions above: incf, decf, multf and divf.

macro
polymorph.maths:incf
place &optional (delta 1)

Like CL:INCF, write a place with its value increased by DELTA. DELTA defaults to NUMBER 1 but can be any type for which the polymorph.maths:+ is defined.

It's possible to update e.g. a setf-able place of type CHARACTER whien delta is of same type too:

(incf (car '(#\1)) #\2) ; => #\c
macro
polymorph.maths:decf
place &optional (delta 1)

Like CL:DECF, write a place with its value decreased by DELTA. DELTA defaults to NUMBER 1.

macro
polymorph.maths:multf
place &optional (delta 1)

Write a place with its value multiplied by DELTA, which defaults to a NUMBER 1.

macro
polymorph.maths:divf
place &optional (delta 1)

Write a place with its value divided by DELTA, which defaults to a NUMBER 1.

Equality & Inequality

polymorphic-function
polymorph.maths:=
object &rest objects

Return T if all of its arguments are equal, in a sense dictated by their type;

NIL otherwise.

Polymorphs:

(= T)

Extend CL:= equality to a single argument of any type. Returns T in this case.

(= NUMBER NUMBER)

(= NULL CONS)

Empty lists (or NIL) and non-empty CONS are not equal.

(= SYMBOL SYMBOL)

(= CHARACTER CHARACTER)

(= STRING STRING)

(= CONS CONS)

(= CONS NULL)

Non-empty and empty CONS (or NIL) are not equal.

(= (AND VECTOR (NOT SIMPLE-ARRAY) (NOT STRING))

(AND VECTOR (NOT SIMPLE-ARRAY) (NOT STRING)))

(= ARRAY ARRAY)

(= HASH-TABLE HASH-TABLE)

(= T T T &REST)

Reduce equality over multiple arguments.

Below are some examples of = applied to the arguments of listed types:

(= 3 3) ; => T

(= "aaa" "bbb") ; => NIL

(= "hello" "hello") ; => T

(= 'a 'A) ; => T

(= '|a| 'A) ; => NIL

(= '(4 . 2) '(4 . 2)) ; => T 

(= '() nil) ; => T

(= '() '(not-empty)) ; => NIL

(= #(1 2 3 4) #(1 2 3 4)) ; => T

(= '(1 1 1 1 1) '(1 1 0 1 1)) ; => NIL
polymorphic-function
polymorph.maths:/=
object &rest objects

Inequality is defined as negation of =.

Meaning it is T when some of its arguments are not =.

Polymorphs:

(/= T)

Extend CL:/= inequality to a single argument of any type. Returns T in this case.

(/= T T)

(/= T T T &REST)

Examples of inequality test:

(/= '(1 1 1 1 1) '(1 1 0 1 1)) ; => T

(/= #(1 2 3 4) #(1 2 3 4)) ; => NIL

(/= 'a '|A| 'A) ; => NIL

Order Comparison Functions

polymorphic-function
polymorph.maths:<
object &rest objects

Return T if its arguments are in strictly increasing order, NIL otherwise.

Polymorphs:

(< T)

(< REAL REAL)

(< CHARACTER CHARACTER)

(< STRING STRING)

(< T T T &REST)

polymorphic-function
polymorph.maths:<=
object &rest objects

Return T if arguments are in strictly non-decreasing order, NIL otherwise.

Polymorphs:

(<= T)

(<= REAL REAL)

(<= CHARACTER CHARACTER)

(<= STRING STRING)

(<= T T T &REST)

polymorphic-function
polymorph.maths:>
object &rest objects

Return T if its arguments are in strictly decreasing order, NIL otherwise.

Defined as negation of polymorph.maths:<=, therefore works for all the same argument types.

By default those include REALs, CHARACTERs and STRINGs.

Polymorphs:

(> T)

(> T T)

(> T T T &REST)

polymorphic-function
polymorph.maths:>=
object &rest objects

Return T if arguments are in strictly non-increasing order, NIL otherwise.

Defined as negation of polymorph.maths:<, therefore works for all the same argument types.

By default those include REALs, CHARACTERs and STRINGs.

Polymorphs:

(>= T)

(>= T T)

(>= T T T &REST)

Parametric-Polymorphic

Functions like min and max that are realized with parametric polymorphism, even if it's more of a convention within this package.

polymorphic-function
polymorph.maths:min
&rest xs

Returns minimum of provided arguments.

Comparison falls back to the polymorph.maths:<

polymorphic order function.

Polymorphs:

(min T)

Obviously returns its single argument when only one is provided.

(min T T)

Returns minimum between FIRST and SECOND argument comparing with

polymorph.maths:<.

(min T T T &REST)

Reduce min over multiple arguments.

polymorphic-function
polymorph.maths:max
&rest xs

Returns maximum of provided arguments.

Comparison falls back to the polymorph.maths:<

polymorphic order function.

Polymorphs:

(max T)

Obviously returns its single argument when only one is provided.

(max T T)

Returns maximum between FIRST and SECOND argument comparing with

polymorph.maths:<.

(max T T T &REST)

Reduce max over multiple arguments.

Comparison works for arguments of all types where < order comparison function is defined.

(min "narrow" "thin" "fat" "thick") ; => "fat"

(min #\c #\, #\; #\a) ; => #\,
